# 🚴‍♂️《元素傳遞》故事文案與 UI/UX 實作完整指南

## 1. 核心設計理念

- **感官體驗優先：** 拒絕長篇大論的維基百科式導覽，將文史故事轉化為感官的延伸。
- **最小必要手機使用：** 考量戶外風切聲與車流聲干擾，放棄語音播放；採用「精煉文字搭配 JRPG 點擊互動」，讓玩家以自己的節奏快速閱讀，掌握主導權後快速回到騎行狀態。

## 2. 站點故事設定 (文案庫)

採用「風之使者」的口吻，將在地歷史小故事與四大元素緊密結合，作為各站拍照任務的引子。

### 📍 站一：香山 17 公里海岸線

- **代表元素：** 風 × 海 (藍色系)
- **在地連結：** 新竹「九降風」與討海人文化。
- **完整文案：**
  > 「感受到了嗎？這是新竹特產的『九降風』。數百年來，這股強風不僅吹乾了米粉與柿餅，更雕塑出這片北台灣最大的濕地。早期討海人就是在這無遮蔽的風勢中，踩著泥濘與海爭食。現在，借用這股形塑大地的力量，**去抓出隱形的風吧！**」

### 📍 站二：通霄 日落大道觀景台

- **代表元素：** 火 (橙色系)
- **在地連結：** 道卡斯族語舊名「吞霄」與落日大景。
- **完整文案：**
  > 「通霄，早期原住民道卡斯族稱這裡為『吞霄』。每當黃昏，巨大的紅日緩緩降入海平線，就像整片海洋張開大口，吞噬了天空中的火球，留下燃燒般的橘紅晚霞。這是一天中最壯烈的時刻，**去尋找屬於你的火焰替身吧！**」

### 📍 站三：大甲 鎮瀾宮前廣場

- **代表元素：** 土 × 木 (大地色系)
- **在地連結：** 先民橫渡黑水溝的實土信仰，與傳承百年的木石雕匠藝。
- **完整文案：**
  > 「兩百多年來，無數先民帶著對未知的恐懼橫渡黑水溝，最終在這裡雙腳踏上『實土』，找到了安定的力量。抬頭看看那些樑柱與石獅，不知名的匠人將畢生心血刻入『木與石』中，更吸收了幾世代人的香火與祈願。**仔細看，去尋找那些阿公級的歲月痕跡。**」

### 📍 站四（終點）：台中舊火車站前廣場

- **代表元素：** 金 (金色系)
- **在地連結：** 1917 年辰野式建築，縱貫線鐵路的黃金年代。
- **完整文案：**
  > 「你抵達了。這座建於 1917 年的紅磚車站，曾是台灣鐵路全線貫通的心臟，見證了無數人相聚與離別的『黃金年代』。百年後，它等到了騎行百公里的你。信封裡其實什麼都沒有，因為真正的『金』，是你一路扛過來的痠痛、汗水與堅持。**現在，留下你淬鍊成金的一句話吧。**」

## 3. UI/UX 介面呈現策略（結合 🎮 JRPG 對話框互動模式）

採用「**滿版卡片式**」設計，透過視覺分層將「閱讀故事」與「執行任務」拆開。為了提升沉浸感並減輕閱讀壓力，第一屏的故事沉浸將全面升級為「JRPG 任務派發模式」，控制在 3 到 4 次點擊內結束。

### 步驟一：第一屏（沉浸故事 —— JRPG 任務派發模式）

- **動態進場：** 玩家掃碼進入網頁時，加入 **1.5 秒的純 CSS `fade-in` 漸顯動畫**。這能模擬「信封慢慢打開」的延遲感，讓玩家沉澱心情準備閱讀。
- **全螢幕沉浸背景：** 依據該站點的元素代表色（如漸層藍、夕陽橘）鋪底，可加入淡淡的動態元素（如微動波浪、閃爍火星），或是直接開啟後置鏡頭的模糊畫面（如果效能允許），讓玩家感覺 NPC 就站在眼前。
- **對話框 UI 佈局：** 畫面下方三分之一處，放置一個帶有半透明黑底、白色邊框的經典 JRPG 對話框。文字排版使用較大的字體、拉大行距，營造短詩或電影台詞的質感。
- **角色頭像（Avatar）：** 對話框左上角或左側，放置「風之使者」的剪影或像素風頭像，增加角色帶入感。
- **引導標示：** 對話框右下角放一個閃爍的向下小箭頭（▼），提示玩家「點擊畫面繼續」。
- **互動節奏（UX）：**
- **打字機特效（Typewriter Effect）：** 文字逐字浮現，營造有人在說話的感覺。速度設定要輕快。
- **防呆與加速：** 考慮到戶外騎車可能很心急，玩家只要「點擊畫面任意處」，就能跳過打字動畫直接顯示整段文字；「再次點擊」則進入下一句對話。

#### 💬 JRPG 腳本拆解範例（以站一：香山為例）

將原本的文案拆解為玩家需要點擊推進的對話陣列（Array），每一段就是一頁對話：

- **[對話 1]** 風之使者：「感受到了嗎？打在臉上的，是新竹特產的『九降風』。」 _(點擊繼續 ▼)_
- **[對話 2]** 風之使者：「數百年來，這股強風不僅吹乾了米粉與柿餅，更雕塑出眼前這片北台灣最大的濕地。」 _(點擊繼續 ▼)_
- **[對話 3]** 風之使者：「早期討海人，就是在這無遮蔽的風勢中，踩著泥濘與海爭食。」 _(點擊繼續 ▼)_
- **[對話 4]** 風之使者：「現在，借用這股形塑大地的力量......」 _(點擊繼續 ▼)_
- _(此時畫面瞬間切換至步驟二的任務模式 UI)_

### 步驟二：第二屏（執行任務）

當對話索引達到陣列最後一項並被點擊時，隱藏 JRPG 對話框元件，渲染出「任務與上傳」介面。

- **介面轉換：** 畫面立刻轉換為俐落、無干擾的操作介面。
- **核心指令：** 特大字體顯示任務目標（例：「**去抓出隱形的風吧！**」📸）。
- **次要提示：** 下方加上小字說明拍攝條件（例：「拍下一張風正在發威或與藍色有關的照片」）。
- **行動呼籲 (CTA)：** 放置一個極其明顯的「開啟相機」或「上傳照片」大按鈕。

## 4. 前端排版與技術實作細節

### Tailwind CSS 視覺優化

為確保戶外強光下的閱讀體驗與整體質感，建議套用以下 Tailwind 實用類別：

- **`tracking-wider`** 或 **`tracking-widest`**：增加字距，讓文字更有呼吸感與排版質感。
- **`text-white/80`**：對於次要文字，使用 80% 不透明度的白色，避免純白字體在陽光下反光過於刺眼，同時拉出資訊層次。

### JRPG 模式前端實作邏輯

在開發上需要新增一個小型的狀態機（State Machine）來控制對話進度：

- **對話索引 (`dialogueIndex`)：** 記錄目前顯示到第幾句話。
- **打字狀態 (`isTyping`)：** 控制玩家點擊時，是要「加速顯示全部文字」還是「進入下一句」。
- **組件切換：** 判斷若 `dialogueIndex` 已達上限，則卸載對話框組件，掛載拍照上傳組件。
